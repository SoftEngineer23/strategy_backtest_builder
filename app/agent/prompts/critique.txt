## Evaluation Rubric

Evaluate against each criterion:

1. ENTRY_RULES_SPECIFIC: Does the strategy have specific, actionable entry rules?
   - PASS: Clear conditions that can be coded (e.g., "RSI crosses below 30")
   - FAIL: Vague conditions (e.g., "enter when price looks oversold")

2. EXIT_RULES_SPECIFIC: Does the strategy have specific, actionable exit rules?
   - PASS: Clear exit conditions including stop loss
   - FAIL: Missing or vague exit rules

3. CONSTRAINTS_ADDRESSED: Does the strategy address the user's constraints?
   - PASS: Each constraint has a corresponding mechanism
   - PARTIAL: Some constraints addressed, others missing
   - FAIL: Constraints ignored
   - N/A: No constraints specified

4. INSTRUMENT_APPROPRIATE: Is the strategy appropriate for the specified instrument(s)?
   - PASS: Strategy accounts for instrument characteristics
   - FAIL: Strategy ignores important instrument factors
   - N/A: No specific instrument mentioned

5. INTERNALLY_CONSISTENT: Is the strategy logic internally consistent?
   - PASS: No contradictions in the rules
   - FAIL: Rules contradict each other

6. IMPLEMENTABLE: Can this strategy be implemented in code?
   - PASS: All rules can be translated to code
   - FAIL: Contains elements that cannot be coded
